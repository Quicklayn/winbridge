# Security Policy

WinBridge is a remote assistance project and has strict abuse-prevention boundaries.

## Supported Scope

Security work currently applies to the bootstrap code, protocol schemas, development relay, OpenSpec requirements, and repository process.

## Prohibited Requests

The project will not implement or accept contributions for:

- Hidden or covert remote sessions.
- Stealth installation.
- Unauthorized persistence.
- Credential theft, keylogging, or secret collection.
- AV/EDR evasion.
- Bypassing Windows security prompts or consent UI.
- Hidden screen capture or hidden remote input.

## Reporting Security Issues

Until a private GitHub Security Advisory workflow is configured, report issues directly to the repository owner.

Include:

- Affected files or components.
- Reproduction steps.
- Expected vs actual behavior.
- Abuse impact.
- Suggested mitigation, if known.

Do not publish exploit details for active vulnerabilities before maintainers have had time to assess and patch them.
